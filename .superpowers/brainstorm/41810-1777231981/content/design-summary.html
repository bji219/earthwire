Phase 2 Design — Earthwire Sample Browser

Confirmed decisions from our brainstorm. Reviewing before writing the spec.

🏗 Architecture
PageStandalone /samples route — separate from the Sequencer, linked from the top nav
LayoutSplit panel — browser (left, flex) + kit builder (right, 300px fixed), always both visible
StateKit state in Svelte store, persisted to localStorage. My Sounds library in IndexedDB (survives refresh).
📂 Sample Browser Tabs (left panel)

📁 My Sounds

Drag-and-drop local audio files. Stored in IndexedDB. Appears first — users build personal libraries over time.

🌿 Freesound

Search Freesound.org API. Nature, ocean, forest, weather. Proxy via /api/freesound to hide API key.

Xeno-canto

Full database search — scientific names, quality filter (A/B/C), country, duration. Raw power-user interface.

🐦 Bird Sounds

Curated Xeno-canto — browse by common name and species family. Simpler for non-birders.

🥁 Kit Builder (right panel — TE drum utility style)
Slots24 numbered rows, each showing pad note (F, F#, G…), sample name, trimmed duration, source badge (xc / fs / local)
Device modeToggle: OP-1/OP-Z (mono, 12s total) or OP-1 field (stereo, 20s total). Changes budget limit and export format.
Time budget barShows total seconds used vs device limit. Turns amber >80%, red if over. Updates live as samples are added/trimmed.
Kit nameEditable inline text field. Used as the exported filename.
KeyboardArrow keys navigate slots · Space previews · Backspace removes · matching TE conventions
✂️ Per-Slot Trim (Option A — waveform expand)
TriggerClick a filled slot → expands inline showing waveform canvas
WaveformGenerated from decoded AudioBuffer, drawn on Canvas. Greyed-out region = trimmed portion.
Trim handleDraggable green handle sets the end point. Text input for precise entry (e.g. "1.4s"). Start offset also adjustable (skip silent intros).
Preview▶ Preview button plays just the trimmed portion via Web Audio
Live updateTime budget bar updates immediately as handle is dragged

⑥ Download .aif
The exported file is a single AIFF with all samples concatenated. An APPL metadata chunk (JSON) tells the OP-1-F where each sample starts/ends — start[], end[], pitch[], volume[], pan[], playmode[] arrays (24 values each).
AIFF encoding runs in a Web Worker so the UI stays responsive during export.
📤 Export Flow
OutputSingle kitname.aif download — no ZIP needed, one file goes straight onto the device
User steps1. Click "export kit" → 2. Download completes → 3. Connect OP-1-F via USB → 4. Drag .aif into SYNTHESIZER/preset/drum/
ValidationExport blocked if total duration exceeds device limit, with a clear error message
