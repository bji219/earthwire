How does the OP-1-F drum kit get to the device?

The export mechanism determines how complex the kit-building backend needs to be.

A

Download Ready-to-Copy ZIP

App generates a single .zip containing the AIFF drum kit file(s) in the correct OP-1-F folder structure. User unzips and drags to device via Finder/Explorer.

Effort: Medium — need to generate AIFF with OP-1 APPL metadata
UX: Simple — works on any computer/browser
Structure: SYNTHESIZER/preset/drum/my-kit.aif
B

Direct USB Transfer (Web USB)

Browser connects directly to the OP-1-F via Web USB API (Chrome only). Files are written straight to the device filesystem — no manual drag needed.

Effort: High — Web USB is low-level, requires MTP/MSC protocol handling
UX: Magical when it works, fragile — Chrome only, may need device in disk mode
Risk: OP-1-F USB protocol not publicly documented
C

Download Individual AIFF + Instructions

Export each sample as a processed .aif file (trimmed, normalized, correct format). User loads them manually into OP-1-F using the drum utility or directly into device folders.

Effort: Lower — no APPL metadata needed, just clean AIFF files
UX: More manual steps, user assembles kit on device
Flexibility: Works with any sampler, not just OP-1-F
